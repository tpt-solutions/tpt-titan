## Security audit

- [x] **Security review (2026-07-17):** Reviewed the backend for common vulnerability classes. Findings:
  - **SQL injection:** No user-controlled string interpolation into SQL found. All dynamic queries use parameterized `db.Query/Exec` with `$N` placeholders; the only string-concatenated SQL fragments (`admin.go` user-deletion loop and `VACUUM ANALYZE`) interpolate `table` names drawn from **hardcoded internal allowlists**, not request input — safe. (Lower risk remains only if those allowlists are ever made caller-supplied.)
  - **Auth/crypto:** Passwords hashed with bcrypt (`golang.org/x/crypto/bcrypt`, `DefaultCost`); JWT secret is injected via `InitAuth(secret)` and the package **panics if used before initialization**, forcing configuration — no compiled-in default secret. AI/speech provider API keys are encrypted at rest (AES-256-GCM, `utils.EncryptPassword`) and marked `json:"-"` so they are never serialized out.
  - **Outstanding honesty items (already labeled, not faked):** biometric face-ID (`utils/crypto.go:227 compareFaceTemplates`) is dormant byte-equality only — must not be exposed without real matching (#68); local/offline STT is honestly "not implemented" per-OS (#66); email virus scanning now honestly records `unverified` when no ClamAV present (#61).
  - **Recommendations (non-blocking, out of this pass's scope):** add `Secure`/`HttpOnly`/`SameSite` attributes to any session cookies if/when cookie auth is introduced; enforce a CSRF token for state-changing form posts; add rate-limiting on `/auth/login`; consider TLS termination guidance in the Caddy/Nginx reverse-proxy docs (already present in `docs/installation.md`).
